# ربط بوابة الدفع — دليل مبسّط

وقت ما تيجيك مفاتيح البوابة (من HyperPay أو Tap أو MyFatoorah)، اعمل الخطوات
دي بالترتيب. كلها في ملف واحد اسمه `.env.local` بجذر المشروع.

## الخطوة ١: افتح (أو أنشئ) ملف .env.local

بجذر مجلد المشروع، أنشئ ملف اسمه بالضبط:  `.env.local`

## الخطوة ٢: اختَر بوابتك وحُطّ مفاتيحها

انسخ القسم الخاص ببوابتك فقط، وحُطّ المفاتيح مكان الفراغات:

### لو اخترت Tap Payments:
```
PAYMENT_PROVIDER=tap
TAP_SECRET_KEY=ضع_المفتاح_السري_هنا
TAP_WEBHOOK_SECRET=ضع_مفتاح_الويبهوك_هنا
```

### لو اخترت HyperPay:
```
PAYMENT_PROVIDER=hyperpay
HYPERPAY_ACCESS_TOKEN=ضع_التوكن_هنا
HYPERPAY_ENTITY_ID=ضع_معرّف_الكيان_هنا
```

### لو اخترت MyFatoorah:
```
PAYMENT_PROVIDER=myfatoorah
MYFATOORAH_API_KEY=ضع_مفتاح_الـAPI_هنا
```

## الخطوة ٣: اربط رابط الإشعار (Webhook) في لوحة البوابة

ادخل على لوحة تحكم البوابة، وحُطّ رابط الإشعار (Webhook URL):
```
https://nnovara.io/api/payments/webhook/tap
```
(بدّل `tap` باسم بوابتك: hyperpay أو myfatoorah)

## الخطوة ٤: أعد تشغيل الموقع
```
npm run dev
```

---

## ⚠️ مهم جداً — للأمان

- **لا تحط المفاتيح السرّية في أي ملف غير `.env.local`.**
- **لا ترفع `.env.local` على GitHub أبداً** (هو محمي تلقائياً).
- المفتاح السري يضل على الخادم فقط — العميل ما بيشوفه إطلاقاً.
- الموقع **ما بيعتبر الطلب مدفوع** إلا لما البوابة نفسها تأكّد الدفع (عبر
  الويبهوك) — مش بمجرّد ما العميل يرجع للصفحة. هاد أمان أساسي ضد التلاعب.

## لسا ما عندك مفاتيح؟

الموقع بيشتغل عادي بدونها — بيعرض للعميل خيار **الحوالة البنكية** و**طلب عرض
سعر** بدل الدفع بالبطاقة. وبمجرّد ما تحط المفاتيح، خيار الدفع بالبطاقة بيظهر
تلقائياً.
